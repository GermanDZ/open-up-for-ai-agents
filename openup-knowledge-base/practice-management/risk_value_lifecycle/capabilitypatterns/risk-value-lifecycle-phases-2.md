Description

This capability pattern illustrates an iterative process with the organization of iterations distributed throughout four phases: [Inception](../guidances/concepts/inception-phase-10.md), [Elaboration](../guidances/concepts/elaboration-phase-10.md), [Construction](../guidances/concepts/construction-phase-10.md), and [Transition](../guidances/concepts/transition-phase-10.md).  Each phase contains one or more iterations. After the last iteration of a phase is performed, there is a milestone that provides the questions that check whether the objectives of that phase were met or not, thereby giving stakeholders and management the opportunity to make business decisions about whether the project should continue to the next phase or not.
