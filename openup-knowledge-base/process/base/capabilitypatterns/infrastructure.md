 In reference to a release sprint, infrastructure refers to all the hardware, software, and network facilities necessary to support a deployed release.
---

Purpose

The purpose of this work product is to provide the underlying capabilities within which an application can be run as designed.

Main Description

Infrastructure normally is defined as anything that supports the flow and processing of information in an organization. The infrastructure needed to support a release package normally includes:
  * Software, including:
    * Operating systems and applications for servers and clients
    * Desktop applications
    * Middleware
    * Protocols
  * Hardware
  * Networks, including:
    * Routers
    * Aggregators
    * Repeaters
    * Other transmission media devices that control movement of data and signals
  * Facilities
