Description

The project starts with the assumption that you have:
  * Created the business case
  * Identified the project manager
  * Defined the team \(at least for the first iteration\)
  * Put in place the development environment \(including tools and infrastructure\)
  * Prepared your organization to follow the [delivery process](../../../core/uma/guidances/termdefinitions/delivery-process.md) suggested by the published process description

The number and length of each Inception iteration will vary depending upon the needs of the project.  The following table summarizes the Inception phase objectives and what activities address each objective:  **Inception phase objectives and activities**

|  **Phase objectives** |  **Activities that address objectives**
---|---
Understand what to build  |  [Identify and Refine Requirements](../capabilitypatterns/identify-and-refine-requirements-1.md)
Identify key system functionality  |  [Initiate Project](../capabilitypatterns/initiate-project-7.md) [Identify and Refine Requirements](../capabilitypatterns/identify-and-refine-requirements-1.md)
Determine at least one possible solution  |  [Agree on Technical Approach](../capabilitypatterns/agree-on-technical-approach-2.md)
Understand the cost, schedule, and risks associated with the project  |  [Initiate Project](../capabilitypatterns/initiate-project-7.md) [Plan and Manage Iteration](../capabilitypatterns/plan-and-manage-iteration-12.md)

Alternatives

This iteration template does not include any activities that result in any implementation. In some projects there will have to be some implementation work done to meet the objectives of Inception. In those cases, additional activities should be added to handle that.
