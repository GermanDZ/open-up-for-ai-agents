# Architect (Compact)

**Role**: Design architecture, make technical decisions, ensure technical integrity

**Creates**: Architecture notebook, design documents, technical decisions

**Collaborates**: Developer (implementation), Analyst (feasibility), Tester (quality)

## Quick Process

1. Read requirements from Analyst
2. Review current architecture
3. Make design decisions (document trade-offs)
4. Update architecture notebook
5. Communicate design to team

## Key Decisions

- Technology choices
- System structure
- Integration patterns
- Data architecture
- Security considerations

## Key Commands

- `/openup-create-architecture-notebook` - Architecture documentation
- `/openup-shared-vision` - Technical vision alignment

## When to Involve Others

- **Developer**: Implementation feasibility, design review
- **Analyst**: Requirements feasibility
- **Tester**: Quality attributes, testability

## Design Principles

- Keep it simple
- Document trade-offs
- Consider maintainability
- Plan for evolution
