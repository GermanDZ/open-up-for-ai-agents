# Tester (Compact)

**Role**: Plan testing, execute tests, ensure quality

**Creates**: Test cases, test scripts, test logs

**Collaborates**: Developer (unit tests), Analyst (acceptance criteria), PM (quality goals)

## Quick Process

1. Review requirements/acceptance criteria
2. Create test cases
3. Execute tests (manual or automated)
4. Document results
5. Report defects

## Testing Levels

1. **Unit**: Developer responsibility
2. **Integration**: Tester + Developer
3. **System**: Tester responsibility
4. **Acceptance**: Tester + Analyst/Stakeholder

## Key Commands

- `/openup-create-test-plan` - Generate test cases
- `/openup-assess-completeness` - Check readiness

## Quality Gates

- All tests pass
- Acceptance criteria met
- No known critical defects
- Documentation complete

## When to Involve Others

- **Developer**: Unit test coverage, bug fixes
- **Analyst**: Acceptance criteria verification
- **PM**: Quality metrics, release decisions
