# Glossary Template

**Purpose**: Maintain a shared vocabulary for consistent terminology across the project. This is especially valuable for AI agents working on projects.

## How to Use This Template

1. Copy this template to `docs/glossary.md` when starting a new project
2. Add domain-specific terms as they are introduced
3. Reference this glossary in other project documents
4. Keep definitions concise and clear

## Template Structure

```markdown
# Glossary

## Domain Terms

| Term | Definition | Related Concepts |
|------|------------|------------------|
| [Term 1] | [Clear definition] | [Links to related terms or concepts] |
| [Term 2] | [Clear definition] | [Links to related terms or concepts] |

## Process Terms

| Term | Definition | Source |
|------|------------|--------|
| [OpenUP Term] | [Definition from OpenUP] | [Link to OpenUP KB] |
```

## Guidelines

- **Be consistent**: Use the same term for the same concept throughout the project
- **Define early**: Add terms when first introduced, not after confusion arises
- **Link to sources**: Reference OpenUP knowledge base for process terms
- **Keep current**: Update definitions if concepts evolve
- **Cross-reference**: Link related terms to help understanding
